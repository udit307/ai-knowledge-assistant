Servlet
     Servlet is an API or interface which is used to communicate between frontend(Html) to Backend(Java prog, database)
-->> Servlet is an interface or API for which implementation is provided by a java program that runs on a servlet container(Tomcat Server)
     and it is used to create dynamic web pages
-->> Servlet is a part od JEE(java enterprise edition) it is responsible for request processing,
     response generation, session management, communication  with database.


Abstract Method that present Inside Servlet Interface
---

interface Servlet {
        void init(ServletConfig);  //-->Use to initialize the servlet/server
        void service(req, resp);   //--> take the request process it and return response
        ServletConfig getServletConfig( );
        String getServletInfo( );
        void destroy( );

}

Types of servlet

1)GenericServlet --> It provides implementation for all the methods present in servlet interface except service method.
                 --> generic servlet is an abstract class that will inherit to servlet interface.
                 --> Genericservlet abstract class is protocol independent and we need to override service( )  method.


2)HttpServlet    --> It is a abstract class which inherits to GenericServlet,
                    it is Specially designed to handle Http specific request.
                    it provide http specific request methods like

 			get : fetch data
 			post : sends data or submits data
 			put : for updating the data
 			delete : for deleting the data

                     --> HttpServlet supports session management.

Difference between Servlet, Generic Servlet, httpServlet

Servlet ; 
it is an interface
it is a packaged in javax.servlet
it is base interface for all types of servlets
it does not support any specific protocol
for all the five abstract method implementation should be provided i.e init( ), getServletConfig( ), getServletInfo( ), service( ), destroy( )
it is difficult to write code and handle the code
it is rarely implemented directly

	

GenericServlet ; 
it is an abstract class
it is a packaged in javax.servlet
it is protocol independent servlet
it does not support any protocol
default implementation is provided, only service( ) method is abstract in nature
it is less difficult to write and maintain the code as compare to servlet
it is implemented for non http protocols like Smtp, Ftp etc.


HttpServlet ; 
it is an abstract class
it is a packaged in javax.servlet.http
it is http specific servlet
it supports http protocol i.e GET, POST, PUT, DELETE ETC.
it provides http specific methods i.e doGet( ), doPost( ), doPut( ), doDelete( ) etc..
it is very easy to write and maintain the code
it is mostly implemented by all the classes or java web application

Diffarence between Forward() method and sendRedirected()

forward( )
-----------
forward() method is present in RequestDispatcher interface.
it will happen at server side.
URL does not change in the browser.
faster in execution.
Data is preserved.
Used for internal navigation.
It is more secure.



sendRedirect( )
---------------
sendRedirect() is present in HttpServlet response.
it happens at client side (Browser Side).
URL changes will be seen in the browser.
Slower in execution.
data is lost.
Used for external redirect or avoid form resubmission.
It is less secure.
